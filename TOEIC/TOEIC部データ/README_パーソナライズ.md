# TOEIC問題パーソナライズシステム

メンバーは**点数だけ入力**すれば、Cursorが自動的に分析してパーソナライズされた問題を生成します。

## 📁 ファイル構成

### 1. `session_results_YYYYMMDD.csv` - セッション結果（メンバーが入力）

**メンバーが入力するのは点数の列だけ！**

```csv
実施日,問題ID,浜脇,中津留,龍生
2025-11-09,148,1,1,0
2025-11-09,149,0,0,1
2025-11-09,150,1,1,0
```

- `1` = 正解
- `0` = 不正解

### 2. `problems_master.csv` - 問題マスタ（Cursorが自動生成）

問題のメタデータを管理します。問題生成時にCursorが自動的に追記します。

```csv
問題ID,問題ファイル,難易度,パッセージ数,トピック,問題タイプ,問題URL
148,148-150.md,初級,1,レストラン,detail,notion.so/xxx
149,148-150.md,初級,1,レストラン,inference,notion.so/xxx
150,148-150.md,初級,1,レストラン,detail,notion.so/xxx
```

## 🔄 ワークフロー

### ステップ1: 問題を生成

```
@問題生成器 フォルダを参照して、以下の条件でTOEIC Part 7の問題を生成してください：
- パッセージ数: 1
- 質問数: 3
- 難易度: 初級
- トピック: レストラン

生成された問題は @問題 フォルダに自動保存し、
問題情報（問題ID、問題ファイル、難易度、パッセージ数、トピック、問題タイプ）を
@TOEIC部データ/problems_master.csv に追記してください。
```

### ステップ2: セッション結果シートを準備

`session_results_template.csv`をコピーして、日付と問題IDを記入：

```csv
実施日,問題ID,浜脇,中津留,龍生
2025-11-09,148,,,
2025-11-09,149,,,
2025-11-09,150,,,
```

### ステップ3: セッション実施（メンバーは点数だけ入力）

各メンバーが自分の列に点数を入力：

```csv
実施日,問題ID,浜脇,中津留,龍生
2025-11-09,148,1,1,0
2025-11-09,149,0,0,1
2025-11-09,150,1,1,0
```

### ステップ4: 分析とパーソナライズ

```
@TOEIC部データ/session_results_20251109.csv と 
@TOEIC部データ/problems_master.csv を参照して、
浜脇さんの以下を分析してください：

1. 全体の正答率
2. 問題タイプ別の正答率（弱点を特定）
3. 難易度別の正答率
4. トピック別の正答率

その上で、弱点を克服するためのパーソナライズされた問題を
@問題生成器 フォルダを参照して生成してください。
```

## 📊 分析例

Cursorが自動的に分析：

```
【浜脇さんの分析結果】
- 全体正答率: 66.7% (2/3問)
- 問題タイプ別:
  - detail: 100% (2/2問)
  - inference: 0% (0/1問) ← 弱点！
- トピック別:
  - レストラン: 66.7%

【推奨される次回の問題】
- 難易度: 初級（現状維持）
- パッセージ数: 1
- 質問数: 5
- トピック: レストラン、ビジネス
- 重点: inference（推論問題）を多めに含める
```

## 💡 メリット

- ✅ **メンバーは点数だけ入力**（1 or 0）
- ✅ 問題のメタデータは自動管理
- ✅ 個人の弱点を自動分析
- ✅ パーソナライズされた問題を自動生成
- ✅ データが蓄積されるほど精度が向上

## 📝 問題タイプ一覧

| 問題タイプ | 説明 |
|-----------|------|
| `detail` | 詳細理解（明示的な情報） |
| `inference` | 推論（暗示的な情報） |
| `vocabulary` | 語彙（単語の意味） |
| `main_idea` | 主旨（文章全体の目的） |
| `NOT` | NOT問題（該当しないもの） |
| `insert_sentence` | 文挿入 |
